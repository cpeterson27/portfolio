# Dashboard Blueprint

Build the dashboard in this order.

## Header

**Notion Builder OS**

Capture signals. Make decisions. Ship useful work. Save what you learn.

Add four callout links:

- Capture an idea
- Start an experiment
- Plan this week
- Save proof

## Row 1 — Product command center

Create a linked Products view:

- Filter: Status is Active
- Layout: gallery or list
- Show: Stage, Current Milestone, Next Decision, Weekly Priority, Blocker, Target Launch

## Row 2 — This week

Create a linked Weekly Shipping view:

- Filter: Status is Active
- Show: Milestone, three outcomes, Smallest Ship, Blockers, Decision Needed
- Keep this section above every backlog

## Row 3 — Decisions and experiments

Use two columns.

Left: linked Decisions view filtered to Status = Proposed or Revisit.

Right: linked Experiments view filtered to Status = Planned or Running.

## Row 4 — Idea pipeline

Create a linked Ideas board:

- Group by Status
- Sort by Priority Score descending
- Hide Deferred and Archived from the main dashboard
- Show Type, Expected Value, Confidence, Effort, Priority Score, and Next Action

## Row 5 — Launch room

Add a page using the Launch Room template. Include linked views for:

- Shipping work connected to the active product
- Experiments connected to launch assumptions
- Approved proof
- Decisions that affect positioning, price, scope, or delivery

## Row 6 — Learning and proof

Use two columns.

Left: recently completed Experiments.

Right: recently captured Proof Library records.

## Footer

Add:

- Weekly review instructions
- Link to the full database library
- Link to the Start Here page
- Product support/contact details

## Design rules

- Keep no more than six top-level dashboard sections.
- Put active work above reference material.
- Default to filtered linked views.
- Hide properties that do not help the current decision.
- Use color only for status and callouts.
- Avoid decorative databases that do not affect product work.
