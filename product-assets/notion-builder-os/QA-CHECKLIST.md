# Product QA Checklist

## Duplication

- [ ] Public template URL opens while signed out
- [ ] Duplicate button is visible
- [ ] Duplicate creates an independent customer copy
- [ ] No private notes, emails, or internal URLs are present

## Dashboard

- [ ] Start Here is visible immediately
- [ ] Active product appears in the command center
- [ ] This Week shows the active shipping record
- [ ] Proposed decisions appear
- [ ] Running experiments appear
- [ ] Idea pipeline filters are correct
- [ ] Proof views show expected records

## Databases

- [ ] Every database has a title property
- [ ] Status options are consistent
- [ ] Product relations work
- [ ] Priority formula handles zero or blank effort
- [ ] Dates display correctly
- [ ] Example records are clearly marked as examples
- [ ] Blank templates do not contain customer-specific information

## Templates

- [ ] Product Brief template creates correctly
- [ ] Idea Review template creates correctly
- [ ] Decision Memo template creates correctly
- [ ] Experiment Brief template creates correctly
- [ ] Weekly Shipping template creates correctly
- [ ] Launch Room template creates correctly
- [ ] Proof Capture template creates correctly

## Customer experience

- [ ] A first-time user understands the four-step workflow
- [ ] Clean Start can be completed in under ten minutes
- [ ] Examples can be deleted without breaking the workspace
- [ ] Mobile dashboard remains readable
- [ ] Support instructions are present
- [ ] Version number and last-updated date are visible

## Commerce readiness

- [ ] Product URL is final
- [ ] Product duplication has been tested
- [ ] Delivery email contains the template link
- [ ] Stripe test purchase completes
- [ ] Success page explains delivery
- [ ] Failed delivery has a support path
- [ ] Refund and support language match the sales page
