# Start Here

Welcome to Notion Builder OS.

This workspace helps you run one simple loop:

**Capture → Decide → Ship → Learn**

## Your first ten minutes

1. Open Products and create a new record using the Product Brief template.
2. Name it and complete the audience, problem, promise, milestone, and next decision. Do not duplicate the example product because its relationships belong to the example workflow.
3. Open Ideas and add one real idea or customer signal.
4. Decide whether the idea should be tested, built, deferred, or archived.
5. Open Weekly Shipping and create the current week.
6. Choose one milestone and the smallest useful thing you can ship.

You are ready. Do not customize every view before using the workflow once.

## The four-step loop

### Capture

Save ideas, customer language, friction, opportunities, and proof before they disappear.

### Decide

Turn important uncertainty into a decision memo or a small experiment.

### Ship

Choose one weekly milestone and visible outcomes. Keep the backlog below active work.

### Learn

Record experiment results, decisions, customer language, screenshots, and outcomes. Feed the useful learning back into the next week.

## Weekly review

Every Friday:

- Mark what shipped.
- Record what did not ship.
- Save useful proof.
- Close or update experiments.
- Capture decisions.
- Choose what rolls into next week.

## Keep it light

Delete views you never use. Hide properties that do not affect decisions. The system should make the work clearer, not become another project to maintain.
